📘 System Technical Explanation

1. JVM Heap Usage
- Phản ánh bộ nhớ dùng bởi Java objects.
- >75%: bắt đầu GC thường xuyên hơn.
- >90% liên tục: nguy cơ Full GC, pause dài, giảm tốc upload.
- Nếu thường xuyên >85% → tăng -Xmx hoặc tối ưu buffer upload.

2. Process CPU
- CPU tiêu thụ bởi tiến trình Java (upload, IO, thread handling).
- >70% liên tục → server đang xử lý nhiều request song song.
- Nếu CPU cao + Thread cao → hệ thống gần quá tải.

3. System CPU
- Tổng CPU toàn hệ thống (bao gồm Docker, OS, service khác).
- Nếu System CPU cao nhưng Process CPU thấp → có tiến trình khác đang chiếm tài nguyên.
- Trong môi trường Docker, có thể bị giới hạn CPU quota.

4. RAM Usage (OS Level)
⚠ RAM usage hiển thị dựa trên số liệu hệ điều hành. Trên Linux, bộ nhớ bao gồm cả page cache và buffer, vì vậy giá trị có thể cao hơn mức thực tế ứng dụng đang sử dụng. Đây chỉ là chỉ số tương đối để theo dõi xu cho vui .
5. Tomcat Thread Pool
- Active Threads: số request đang xử lý.
- Max Threads: giới hạn cấu hình connector.
- >80% thread usage → request mới có thể phải chờ.
- 100% + Queue tăng → bắt đầu delay upload.

6. Queue Size
- Số request đang chờ thread xử lý.
- Queue >0 thường xuyên → hệ thống đang vượt khả năng xử lý tức thời.
- Nếu queue tăng dần theo thời gian → cần tăng maxThreads hoặc scale server.

7. Connection Timeout
- Nếu upload file lớn và timeout thấp → có thể bị cắt giữa chừng.
- Với upload LAN nên >= 60000ms.

⚠ Tình huống quá tải thường gặp khi upload:
- Nhiều file lớn upload song song → thread pool full
- File >5GB + WiFi yếu → giữ thread lâu → queue tăng
- Disk IO chậm → CPU thấp nhưng thread vẫn cao

Khuyến nghị vận hành:
- Giữ Thread usage < 75% khi chạy production
- Giữ Heap usage < 85%
- Monitor queue nếu upload > 20 user cùng lúc
- Nếu Docker: đảm bảo container không bị giới hạn RAM/CPU
